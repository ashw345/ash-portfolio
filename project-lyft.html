Lyft 街景模式

解决用户在打车过程中难以找到司机和上车点的问题，让打车更简单。

为 Lyft 网约车进行迭代设计，分析当前版本问题，并根据问题和目标人群定制创新解决方案。本项目由我独立完成，是未落地的学生项目。

我负责 用户研究 / AB测试 / UI设计
项目类型 学校项目
时长 2023.09 — 2023.10
工具 Figma
Problem

现有问题分析

乘客在打车时难以准确地找到司机和上车点的位置。在繁忙路口，周边车辆太多难以辨认；在陌生街道，不了解周边环境导致迷失。

场景一：繁忙路口

场景二：陌生街道

场景三：地点模糊

Goal

优化乘客找到上车点的路径，并且使用户更了解司机的位置。

Exploration

配色方案探索

在项目初期，我们探索了三套不同的配色方向，从能量感的粉色系，到专业稳重的蓝色系，再到温暖友好的橙色系，最终选择了粉色作为品牌主色，以传达安全、即时和活力的设计理念。

Design System

视觉设计规范

围绕"清晰、安全、即时"的设计理念，构建了一套以高对比度黑白为主、辅以品牌粉色作为关键交互色的视觉系统。强调信息层级的清晰可读与移动场景下的快速识别。

解决方案 Solution 01

司机位置街景

当司机前往上车点时，用户可以在街景中看到司机汽车模型的精确位置，车辆下方显示进行路径，让用户了解司机来的具体方向和停车位置。

解决方案 Solution 02

上车点街景导航

用户可以通过街景查看上车点周边的实际环境，配合指引箭头步行导航，减少在陌生地点的迷失感。
